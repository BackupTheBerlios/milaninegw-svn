Warning

These upgrade instructions apply to an upgrade from Vanilla 0.9.0 (or greater) to 0.9.2. If you have a different version of Vanilla installed, following these instructions may break your existing installation. We highly recommend that you back up your database and files before you attempt to perform these changes.

Database Changes

There was a minor change made to the database in this release, so you will need to run the following SQL script on your database:

alter table LUM_Role add CanViewIps enum('1','0') not null default '0';

File Updates

Every single file in Vanilla was changed in this upgrade. As a result, you will need to manually make changes to a couple of small files, and over-write all other files.

File Changes

languages/English.php
If you have altered your languages/English.php file, or created your own custom language file, rather than overwrite your customized language file with our new version, compare with our new version. All changes made to the language file are appended and documented at the end of the file. Make sure that any new codes are copied over into your existing language file.
Change the very first line in the file from <? to <?php.

appg/headers.php
Change the first line from <? to <?php.
appg/settings.php
Change the first line from <? to <?php.
Add the following code below line 32: define("agDISCUSSIONS_PER_FEED", "20");.
Upgrade your version number on line 85 from 0.9.1 to 0.9.2.

File Replacements

The following files and folders can be completely overwritten with the new versions:

  • appg/init_ajax.php
  • appg/init_external.php
  • appg/init_internal.php
  • The entire /controls directory
  • The entire /extensions directory
  • The entire /feeds directory
  • The entire /images directory
  • The entire /js directory
  • The entire /library directory
  • The entire /styles directory
  • The entire /tools directory
  • All files in the base directory (account.php, apply.php, etc)
